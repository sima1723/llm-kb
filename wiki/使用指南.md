# LLM 知识库 — 完整使用指南

> 核心理念（来自 Andrej Karpathy）：知识不只是记录，更要**可搜索、可追问、可互相关联**。
> LLM 充当"研究图书馆员"——主动编译、链接、维护 Markdown 文件，不依赖向量数据库。

---

## 一、完整工作闭环

```
原始资料（PDF / 网页 / 视频 / 手写笔记）
        ↓  make clip / clip-video / extract / note
    raw/ 目录（Markdown 原始文件）
        ↓  make compile
    wiki/ 目录（结构化知识条目 + INDEX.md）
        ↓              ↑
  make search        make explore-add + stub-fill
  make ask-save  →  wiki/answers/ → 被搜索到（答案回流）
        ↓
  make slides / report / brief（输出）
        ↓
  make compile-commit（git 快照，知识演进可追溯）
```

---

## 二、第一步：初始化项目

```bash
git clone <repo-url> llm-kb
cd llm-kb
export ANTHROPIC_API_KEY=sk-ant-...   # 填入你的 API Key
make init                              # 安装依赖，创建目录，验证连通性
```

初始化后目录结构：

```
llm-kb/
├── raw/           # 放原始资料
│   ├── articles/  # 网页文章
│   ├── papers/    # 论文 PDF
│   ├── media-notes/ # 视频/播客
│   └── repos/     # 代码仓库笔记
├── wiki/          # 编译后的知识条目
├── templates/     # LLM prompt 模板
├── tools/         # 工具代码
└── config.yaml    # 配置文件
```

## 三、第二步：摄入原始资料

### 3.1 抓取网页文章

```bash
make clip URL=https://karpathy.ai/zero-to-hero.html
# → 保存到 raw/articles/YYYY-MM-DD-标题.md
```

### 3.2 提取 PDF 论文

```bash
make extract PDF=raw/papers/attention_is_all_you_need.pdf
# → 保存到 raw/papers/attention_is_all_you_need.md
```

### 3.3 提取 YouTube / 视频字幕（需 yt-dlp）

```bash
pip install yt-dlp                     # 一次性安装
make clip-video URL=https://youtube.com/watch?v=xxx
# → 优先下载字幕，保存到 raw/media-notes/YYYY-MM-DD-标题.md

make clip-video URL=... WHISPER=1      # 字幕不可用时用 Whisper 转录
# 需要额外安装：pip install openai-whisper
```

### 3.4 新建手动笔记

```bash
make note TITLE="Transformer 论文精读" TYPE=article
make note TITLE="No Priors 播客 S03E12" TYPE=media-note
# → 生成带 frontmatter 的笔记模板，手动填写后再编译
```

> **何时用哪种**：网页 → `clip`，论文 PDF → `extract`，视频 → `clip-video`，读书/听课笔记 → `note`

## 四、第三步：编译成 wiki 知识条目

### 4.1 增量编译（最常用）

```bash
make compile
# 只处理 raw/ 中新增或修改的文件，已处理的跳过
# 结束后自动重建 INDEX.md
```

### 4.2 预览将要编译什么（不花钱）

```bash
make compile-dry
```

### 4.3 全量重编译

```bash
make compile-full
# 清空状态，重新处理所有 raw/ 文件
```

### 4.4 编译 + 自动 git 提交（知识演进可追溯）

```bash
make compile-commit
# 等价于：make compile，结束后 git add wiki/ && git commit
```

也可在 `config.yaml` 中永久开启：

```yaml
compile:
  git_auto_commit: true
```

### 4.5 费用控制

`config.yaml` 中设置单次编译上限，超出后自动停止：

```yaml
compile:
  budget_limit_usd: 5.0
```

编译 1 篇中等文章约 $0.05，10 篇约 $0.30-0.80。

## 五、第四步：搜索与问答

### 5.1 关键词搜索（TF-IDF，离线瞬时）

```bash
make search Q="注意力机制"
make search Q="scaling law"
```

适合：已知关键词，想快速定位条目。

### 5.2 语义搜索（LLM 理解查询意图）

```bash
make search-semantic Q="Self-Attention 是怎么工作的"
make search-semantic Q="LLM 为什么需要位置编码"
```

适合：跨语言查询（Self-Attention → 自注意力）、近义词、概念级问题。

### 5.3 基础问答

```bash
make ask Q="什么是涌现能力？"
# LLM 用 wiki 内容回答，不保存
```

### 5.4 问答 + 答案回流（越问越智能）

```bash
make ask-save Q="Transformer 和 RNN 的核心区别是什么？"
# 回答保存到 wiki/answers/，下次搜索时会检索到
```

### 5.5 深度问答（递归读取关联条目）

```bash
make ask-deep Q="解释 RLHF 的完整训练流程"
# 找到相关条目后，再读取这些条目的关联条目，提供更丰富的上下文
```

> **答案回流机制**：`ask-save` 保存的答案存入 `wiki/answers/`，
> 之后 `make search` 会检索到它，`make ask` 也会把它作为上下文。
> 知识库越用越智能。

## 六、第五步：知识库维护

### 6.1 健康检查

```bash
make lint           # 检查断链、孤立条目、格式问题
make lint-fix       # 自动修复本地可修复的问题（空条目、INDEX 不同步）
make lint-ai        # 调用 LLM 修复复杂问题（内容矛盾、描述不准确）
```

### 6.2 探索建议 → 创建 stub → 填充 stub（知识扩展闭环）

```bash
# 第一步：让 LLM 分析知识库，找出值得深入研究的方向和未收录的概念
make explore

# 第二步：自动为建议的概念创建空 stub 条目
make explore-add

# 第三步：让 LLM 用训练知识填充这些 stub 条目（Karpathy 核心理念）
make stub-fill                    # 填充所有 stub
make stub-fill ENTRY=规模定律     # 只填充一个
make stub-fill-dry                # 预览待填充列表
```

> 这是 Karpathy 方案的精髓：**LLM 不只是检索工具，更是主动编撰者**。
> `explore-add` + `stub-fill` 让知识库可以自我扩展，不依赖手动输入新资料。

### 6.3 统计仪表盘

```bash
make stats          # 条目数、字数、来源数、关联密度等统计
make status         # 快速状态：待处理 raw 文件数 + 条目总数
```

## 八、典型使用场景

### 场景 A：读完一篇论文，加入知识库

```bash
make extract PDF=raw/papers/my-paper.pdf
make compile
make ask-save Q="这篇论文的核心创新点是什么？"
```

### 场景 B：看完一个 YouTube 讲座

```bash
make clip-video URL=https://youtube.com/watch?v=xxx
make compile
make slides TOPIC="讲座核心主题"
```

### 场景 C：扩充知识库中的薄弱领域

```bash
make explore        # 看看哪些方向需要深入
make explore-add    # 创建建议概念的 stub
make stub-fill      # LLM 填充初稿
make compile-commit # 编译新资料（如果有）并 git 快照
```

### 场景 D：定期维护

```bash
make lint-fix       # 修复格式问题
make lint-ai        # 修复内容问题
make stats          # 查看知识库增长情况
```

## 九、常见问题

**Q: 编译失败了怎么办？**
单个文件失败不影响其他文件。失败记录在 `.state/`，再次运行 `make compile` 自动重试。

**Q: stub 条目和普通条目的区别？**
stub 条目（`tags: [stub]` 或含"待补充"）是占位符，可用 `make stub-fill` 让 LLM 填充。
填充后建议通过 `make clip/extract` 补充原始资料，再 `make compile` 精化内容。

**Q: 费用超出了怎么办？**
降低 `config.yaml` 中的 `budget_limit_usd`，或换用更便宜的模型（改 `llm.model`）。

**Q: 可以与 Obsidian 一起用吗？**
可以。用 Obsidian 打开 `wiki/` 目录，`[[链接]]` 自动渲染为双向链接图谱。
`wiki/slides/` 中的幻灯片可用 Obsidian Marp 插件预览。

**Q: 想追溯知识库的历史变化？**
用 `make compile-commit` 或在 `config.yaml` 设 `git_auto_commit: true`，
每次编译都会生成 git commit，`git log wiki/` 可查看每个条目的演化历史。
